# คู่มือการใช้งาน Section สำหรับแบบฟอร์ม (ภาษาไทย)

## ภาพรวม
ฟีเจอร์ Section ช่วยให้คุณจัดกลุ่มฟิลด์ในแบบฟอร์มเป็นส่วนๆ ทำให้แบบฟอร์มเป็นระเบียบและง่ายต่อการทำมากขึ้น

## วิธีใช้งาน

### 1. การเพิ่ม Section ใหม่

1. ไปที่หน้าสร้างแบบฟอร์ม `/admin/forms/new`
2. ที่แผงด้านซ้ายมือ ด้านล่างของประเภทฟิลด์ จะมีปุ่ม **"+ เพิ่มส่วนใหม่"**
3. คลิกเพื่อเพิ่ม Section ใหม่

### 2. การตั้งค่า Section

เมื่อคลิกที่ส่วนหัวของ Section (พื้นที่สีเขียว) จะเปิดหน้าจอตั้งค่า:

- **ชื่อส่วน** - ชื่อที่แสดงให้ผู้ตอบแบบฟอร์มเห็น (เช่น "ข้อมูลทั่วไป", "ข้อมูลส่วนบุคคล")
- **คำอธิบายส่วน** - คำอธิบายเพิ่มเติมที่แสดงใต้ชื่อส่วน
- **จำนวนฟิลด์** - แสดงจำนวนฟิลด์ในส่วนนี้ (อ่านอย่างเดียว)
- **ลำดับที่** - แสดงตำแหน่งของ Section (อ่านอย่างเดียว)

### 3. การเพิ่มฟิลด์ลงใน Section

มี 2 วิธี:

**วิธีที่ 1: เพิ่มผ่านปุ่มลัด**
- แต่ละ Section จะมีปุ่มลัดประเภทฟิลด์ 8 ประเภทที่ด้านล่าง
- คลิกที่ไอคอนเพื่อเพิ่มฟิลด์ทันที

**วิธีที่ 2: เพิ่มจากแผงด้านซ้าย**
- คลิกที่ประเภทฟิลด์จากแผงด้านซ้าย
- ฟิลด์จะถูกเพิ่ม vào Section แรกโดยอัตโนมัติ

### 4. การย้ายฟิลด์ระหว่าง Sections

ใช้ Dropdown "ย้ายไป" ที่出现在每个ฟิลด์:
- คลิกที่ Dropdown
- เลือก Section ที่ต้องการย้าย
- ฟิลด์จะถูกย้ายทันที

### 5. การจัดการ Sections

**ย้ายตำแหน่ง Section:**
- ใช้ปุ่ม ⬆️ เพื่อย้ายขึ้น
- ใช้ปุ่ม ⬇️ เพื่อย้ายลง

**คัดลอก Section:**
- คลิกปุ่ม 📋
- Section พร้อมฟิลด์ทั้งหมดจะถูกคัดลอก

**แก้ไข Section:**
- คลิกที่ส่วนหัวของ Section
- แก้ไขชื่อและคำอธิบาย
- คลิก "เสร็จสิ้น"

**ลบ Section:**
- คลิกปุ่ม 🗑️
- Section และฟิลด์ทั้งหมดในส่วนจะถูกลบ

### 6. การแสดงผลในแบบฟอร์ม

เมื่อผู้ตอบแบบฟอร์มเปิดแบบฟอร์มที่มี Sections:

```
┌─────────────────────────────────────┐
│  ชื่อแบบฟอร์ม                       │
│  คำอธิบายแบบฟอร์ม                   │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  📁 ส่วนที่ 1: ข้อมูลทั่วไป         │
│  กรุณากรอกข้อมูลพื้นฐาน             │
├─────────────────────────────────────┤
│  • ชื่อ-นามสกุล                     │
│  • อีเมล                            │
│  • เบอร์โทร                         │
└─────────────────────────────────────┘

───────────────────────────────────────

┌─────────────────────────────────────┐
│  📁 ส่วนที่ 2: ข้อมูลส่วนบุคคล      │
│  ข้อมูลเพิ่มเติมเกี่ยวกับคุณ         │
├─────────────────────────────────────┤

│  • ที่อยู่                          │

```

## ตัวอย่างการใช้งานจริง

### แบบฟอร์มลงทะเบียนงานสัมมนา

**ส่วนที่ 1: ข้อมูลส่วนตัว**
- ชื่อ-นามสกุล (Text, Required)
- อีเมล (Email, Required)
- เบอร์โทร (Tel)
- หน่วยงาน (Select)

**ส่วนที่ 2: ข้อมูลการเข้าร่วม**
- ชื่อสัมมนาที่สมัคร (Radio)
- จำนวนคนที่มาด้วย (Number)
- อาหารที่แพ้ (Checkbox)
- ความต้องการพิเศษ (Textarea)

**ส่วนที่ 3: การชำระเงิน**
- วิธีการชำระเงิน (Select)
- อัพโหลดสลิปการโอน (File)
- หมายเหตุ (Textarea)

### แบบฟอร์มสำรวจความพึงพอใจ

**ส่วนที่ 1: ข้อมูลผู้ตอบ**
- เพศ (Radio)
- อายุ (Select)
- หน่วยงาน (Select)

**ส่วนที่ 2: การประเมิน**
- ความพึงพอใจต่อบริการ (Radio 1-5)
- ความพึงพอใจต่อสิ่งอำนวยความสะดวก (Radio 1-5)
- ความพึงพอใจต่อเจ้าหน้าที่ (Radio 1-5)

**ส่วนที่ 3: ข้อเสนอแนะ**
- สิ่งที่ชอบ (Textarea)
- สิ่งที่ควรปรับปรุง (Textarea)
- ข้อเสนอแนะเพิ่มเติม (Text)

## โครงสร้าง JSON Schema

เมื่อใช้ Sections โครงสร้าง JSON จะเปลี่ยนเป็น:

```json
{
  "sections": [
    {
      "id": "section-uuid-1",
      "title": "ข้อมูลทั่วไป",
      "description": "กรุณากรอกข้อมูลพื้นฐาน",
      "fields": ["field-uuid-1", "field-uuid-2", "field-uuid-3"]
    },
    {
      "id": "section-uuid-2",
      "title": "ข้อมูลส่วนบุคคล",
      "description": "ข้อมูลเพิ่มเติม",
      "fields": ["field-uuid-4", "field-uuid-5"]
    }
  ],
  "fields": [
    {
      "id": "field-uuid-1",
      "type": "text",
      "label": "ชื่อ-นามสกุล",
      "required": true
    },
    {
      "id": "field-uuid-2",
      "type": "email",
      "label": "อีเมล",
      "required": true
    }
    // ... ฟิลด์อื่นๆ
  ],
  "settings": {
    "theme_color": "#3B82F6",
    "allow_multiple_submissions": true
  }
}
```

## Backward Compatibility

ระบบรองรับแบบฟอร์มเก่าที่ไม่มี Sections:
- แบบฟอร์มเดิมจะยังแสดงผลได้ปกติ
- ฟิลด์ทั้งหมดจะแสดงเรียงต่อกันโดยไม่มี Section
- สามารถเพิ่ม Sections ใหม่ให้แบบฟอร์มเดิมได้

## เคล็ดลับการใช้งาน

1. **ตั้งชื่อ Section ให้ชัดเจน** - ผู้ตอบจะได้รู้ว่าต้องกรอกอะไร
2. **ใช้คำอธิบาย** - ช่วยให้ผู้ตอบเข้าใจวัตถุประสงค์ของแต่ละส่วน
3. **อย่าแบ่งส่วนเยอะเกินไป** - 3-5 ส่วนกำลังดี
4. **จัดกลุ่มฟิลด์ที่เกี่ยวข้องกัน** - ทำให้แบบฟอร์มเป็นตรรกะ
5. **ใช้ Section กับแบบฟอร์มยาว** - แบบฟอร์มสั้นอาจไม่จำเป็นต้องแบ่ง

## การแก้ปัญหา

### Section ไม่แสดง
- รีเฟรชหน้าเว็บ
- ตรวจสอบว่าเพิ่ม Section แล้วจริงๆ
- เคลียร์ Cache ของ Browser

### ฟิลด์หายหลังจากย้าย Section
- ตรวจสอบว่าฟิลด์ยังอยู่ใน array `fields` ของ Schema
- ลองบันทึกแบบฟอร์มใหม่

### ไม่สามารถย้ายฟิลด์ได้
- ตรวจสอบว่ามีอย่างน้อย 2 Sections
- Dropdown อาจมีปัญหา Browser - ลองเคลียร์ Cache

✅ ทำให้แบบฟอร์มดูเป็นระเบียบ
✅ เพิ่มความง่ายในการทำแบบฟอร์ม
✅ ปรับปรุง User Experience
