# Form Access Management Menu - การเพิ่มเมนูจัดการสิทธิ์

## สรุป

ได้เพิ่มเมนู **"อนุญาตให้เข้าถึงข้อมูลการตอบ"** (Form Access Management) ใน Admin Panel เพื่อให้เจ้าของแบบฟอร์มสามารถจัดการคำขอสิทธิ์และมอบสิทธิ์เข้าถึงข้อมูลการตอบแบบฟอร์มได้ง่ายๆ

## ไฟล์ที่สร้าง/แก้ไข

### 1. สร้างหน้าใหม่
**File:** `resources/js/pages/admin/FormAccessManagement.jsx`

หน้าจัดการสิทธิ์ที่มีคุณสมบัติ:
- ✅ เลือกแบบฟอร์มที่ต้องการจัดการ
- ✅ ดูคำขอสิทธิ์ที่รออนุมัติ
- ✅ อนุมัติคำขอแบบเดี่ยวหรือแบบชุด
- ✅ ปฏิเสธคำขอพร้อมเหตุผล
- ✅ มอบสิทธิ์โดยตรง (ไม่ต้องรอคำขอ)
- ✅ ดูรายชื่อผู้ใช้ที่มีสิทธิ์
- ✅ เพิกถอนสิทธิ์
- ✅ ดูประวัติ Audit Log
- ✅ กำหนดวันหมดอายุสิทธิ์

### 2. เพิ่ม Route
**File:** `resources/js/routes.jsx`

```javascript
<Route path="form-access" element={<FormAccessManagement />} />
```

**URL:** `/admin/form-access`

### 3. เพิ่มเมนูใน Sidebar
**File:** `resources/js/components/Layouts/AdminLayout.jsx`

เพิ่มเมนู:
```jsx
<li>
  <Link to="/admin/form-access">
    <i className="fas fa-key"></i>
    <span>อนุญาตให้เข้าถึงข้อมูลการตอบ</span>
  </Link>
</li>
```

## การใช้งาน

### 1. เข้าหน้าจัดการสิทธิ์
- คลิกเมนู **"อนุญาตให้เข้าถึงข้อมูลการตอบ"** ใน Admin Panel
- เลือกแบบฟอร์มที่ต้องการจัดการจาก dropdown

### 2. ดูคำขอที่รออนุมัติ (Tab: รออนุมัติ)
- แสดงรายการคำขอทั้งหมดที่รอการอนุมัติ
- ข้อมูลที่แสดง:
  - ชื่อ-อีเมล ผู้ขอ
  - ระดับสิทธิ์ที่ขอ
  - เหตุผลที่ขอ
  - ปุ่มดำเนินการ (อนุมัติ/ปฏิเสธ)

### 3. อนุมัติคำขอ
**อนุมัติแบบเดี่ยว:**
- คลิกปุ่ม "อนุมัติ (ดู)", "อนุมัติ (ส่งออก)", หรือ "อนุมัติ (จัดการ)"
- ระดับสิทธิ์จะถูกระบุตามปุ่มที่เลือก

**อนุมัติแบบชุด:**
- คลิกปุ่ม "อนุมัติทั้งหมด"
- ทุกคำขอที่รอจะได้รับการอนุมัติด้วยสิทธิ์ "ดูผลลัพธ์"

### 4. ปฏิเสธคำขอ
- คลิกปุ่ม "ปฏิเสธ"
- ระบุเหตุผลที่ปฏิเสธ
- คำขอจะถูกบันทึกพร้อมเหตุผล

### 5. มอบสิทธิ์โดยตรง (Tab: อนุมัติแล้ว)
- คลิกปุ่ม "มอบสิทธิ์โดยตรง"
- กรอกข้อมูล:
  - User ID ของผู้ใช้
  - ระดับสิทธิ์ (ดู/ส่งออก/จัดการ)
  - วันหมดอายุ (ถ้าต้องการ)
  - หมายเหตุ
- คลิก "มอบสิทธิ์"

### 6. เพิกถอนสิทธิ์
- ไปที่ Tab "อนุมัติแล้ว"
- คลิกปุ่ม "เพิกถอนสิทธิ์" ของผู้ใช้ที่ต้องการ
- ยืนยันการเพิกถอน

### 7. ดูประวัติ (Tab: ประวัติ)
- แสดง Audit Log ทุกการดำเนินการ
- ข้อมูลที่แสดง:
  - การกระทำ (อนุมัติ/ปฏิเสธ/เพิกถอน/มอบสิทธิ์)
  - ผู้กระทำ
  - เหตุผล
  - วันที่เวลา

## ระดับสิทธิ์

| ระดับ | คำอธิบาย | สิทธิ์ |
|-------|----------|--------|
| ดูผลลัพธ์ | `view` | ดูข้อมูลการตอบ |
| ดูและส่งออกข้อมูล | `export` | ดู + ส่งออกเป็น CSV |
| จัดการทั้งหมด | `manage` | ดู + ส่งออก + จัดการ (เพิ่มโน้ต, ลบ) |

## API Endpoints ที่ใช้

| Endpoint | Method | คำอธิบาย |
|----------|--------|----------|
| `/api/forms` | GET | ดึงรายการแบบฟอร์ม |
| `/api/forms/{uuid}/access/requests/pending` | GET | ดูคำขอที่รอ |
| `/api/forms/{uuid}/access/requests` | GET | ดูคำขอทั้งหมด |
| `/api/forms/{uuid}/access/users` | GET | ดูผู้ใช้ที่มีสิทธิ์ |
| `/api/forms/{uuid}/access/audit-log` | GET | ดูประวัติ |
| `/api/forms/{uuid}/access/requests/{id}/approve` | POST | อนุมัติคำขอ |
| `/api/forms/{uuid}/access/requests/{id}/reject` | POST | ปฏิเสธคำขอ |
| `/api/forms/{uuid}/access/requests/batch` | POST | อนุมัติแบบชุด |
| `/api/forms/{uuid}/access/grant` | POST | มอบสิทธิ์โดยตรง |
| `/api/forms/{uuid}/access/users/{userId}` | DELETE | เพิกถอนสิทธิ์ |

## ตัวอย่างการใช้งาน

### อนุมัติคำขอ
```
1. เจ้าของแบบฟอร์มเข้าเมนู "อนุญาตให้เข้าถึงข้อมูลการตอบ"
2. เลือกแบบฟอร์มที่ต้องการจัดการ
3. เห็นคำขอจาก "สมชาย" ขอสิทธิ์ "ดูผลลัพธ์"
4. คลิก "อนุมัติ (ดู)"
5. สมชายได้รับสิทธิ์และสามารถดูข้อมูลได้
```

### มอบสิทธิ์โดยตรง
```
1. เจ้าของแบบฟอร์มต้องการมอบสิทธิ์ให้ "สมหญิง"
2. คลิก "มอบสิทธิ์โดยตรง"
3. กรอก User ID ของสมหญิง
4. เลือกระดับสิทธิ์ "จัดการทั้งหมด"
5. กำหนดวันหมดอายุ 31 ธ.ค. 2026
6. คลิก "มอบสิทธิ์"
7. สมหญิงได้รับสิทธิ์ทันที
```

## การแจ้งเตือน

ระบบจะส่งการแจ้งเตือนไปยังผู้ใช้ในกรณี:
- คำขอสิทธิ์ได้รับการอนุมัติ
- คำขอสิทธิ์ถูกปฏิเสธ
- ได้รับสิทธิ์โดยตรง
- สิทธิ์ถูกเพิกถอน

## หมายเหตุ

- **เจ้าของแบบฟอร์ม** เท่านั้นที่สามารถเข้าหน้านี้ได้
- **Internal Users** (@pcru.ac.th) ได้สิทธิ์ `manage` อัตโนมัติ
- สิทธิ์ที่หมดอายุจะถูกตรวจสอบอัตโนมัติ
- ทุกการดำเนินการถูกบันทึกใน Audit Log
